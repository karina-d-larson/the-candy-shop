Project: The Candy Shop Wars Catalog

Project Title

The Candy Shop Wars Catalog

Project Type

Storefront

Project Description

The Candy Shop Wars Catalog is a storefront-style web application inspired by the Candy Shop Wars book series. Users will be able to browse a collection of magical candies and items, filter items by category, view detailed item information, save favorite items, and submit an order form.

The project will keep the same structure as the storefront assignment while using a custom theme. The goal is to create a simple, accessible, and responsive online catalog that feels magical and adventurous without becoming too complicated.

Color Palette

The color palette is inspired by the Candy Shop Wars trilogy covers, especially the purple box set, teal/green magical lighting, and gold title lettering.

Color Role Color Name Hex Code Use
Primary Deep Purple #5B2A86 Header, navigation, major branding
Secondary Deep Teal #1F5F67 Buttons, links, secondary UI elements
Accent Antique Gold #D4A54A Borders, badges, highlights, decorative details
Background Warm Cream #F8F4EC Main page background
Text Dark Cocoa #2D221B Main body text

To support accessibility, gold will mainly be used as a decorative accent rather than for important text. Main text will use dark cocoa on a warm cream background, and buttons will use white text on dark purple or teal backgrounds for stronger contrast.

Typography

Use Font
Headings Cinzel
Body Text Inter

Cinzel will be used for headings because it gives the site a magical, book-inspired feeling. Inter will be used for body text because it is clean, readable, and works well for forms, product cards, and longer descriptions.

Planned Pages / Views

Home / Catalog Page

The home page will display the main product catalog. It will include a header, navigation, a short intro section, a category filter dropdown, and a responsive grid of magical item cards.

Each item card will include:

  • Item image
  • Item name
  • Category
  • Price
  • Short description
  • View Details button

Item Detail Page

The item detail page will show more information about one selected catalog item. The correct item will load using a URL parameter, such as ?id=moon-rocks.

The detail page will include:

  • Larger item image
  • Item name
  • Price
  • Category
  • Rarity
  • Full description
  • Magical effect
  • Add to Favorites button
  • Order button
  • Back to Catalog link

Order Form Page

The order form page will allow users to submit a mock order request for an item. The selected item will be shown in an order summary area.

The form will include:

  • Name
  • Email
  • Phone
  • Address
  • Optional message

Required fields will be validated before submission.

Favorites Page

The favorites page will show items the user saved. Favorites will be stored in localStorage so they remain after the page refreshes.

The page will include:

  • Saved item cards
  • Remove favorite button
  • Empty state message if no favorites have been saved

Planned JavaScript Modules

Module Purpose
main.js Main entry file. Handles page setup, event listeners, and calls functions from other modules.
data.js Fetches item data from the local JSON file and handles loading/error states.
render.js Contains functions for rendering item cards, item details, favorites, and order summaries.
storage.js Handles localStorage features, including saving favorites and tracking order count.
utils.js Contains helper functions, such as getting URL parameters or formatting prices.

Planned JSON Structure

Product data will be stored in a local JSON file named items.json. The file will include at least 12 magical candies or items.

Example item structure:

{
  "id": "moon-rocks",
  "name": "Moon Rocks",
  "category": "Magical Candy",
  "price": 6.99,
  "rarity": "Rare",
  "shortDescription": "A mysterious candy with unusual movement-based effects.",
  "description": "Moon Rocks are magical candies known for giving the user a strange and powerful ability for a short time.",
  "magicalEffect": "Allows temporary gravity-defying movement.",
  "image": "images/moon-rocks.jpg"
}

Planned item categories may include:

  • Magical Candy
  • Utility Item
  • Artifact
  • Carnival Item
  • Adventure Gear

Planned localStorage Feature

The main localStorage feature will be saved favorites. When users click “Add to Favorites,” the item ID will be saved in localStorage. When they return to the site or refresh the page, their favorite items will still appear.

Form Validation Plan

The order form will collect the following information:

  • Name, required
  • Email, required
  • Phone number, required
  • Address, required
  • Optional message

Validation rules:

  • Name must not be empty.
  • Email must use a valid email format.
  • Phone number must contain a valid phone number format.
  • Address must not be empty.
  • Required fields will display clear error messages if validation fails.
  • Invalid fields will be visually identified without relying on color alone.
  • Error messages will be associated with their corresponding inputs for accessibility.

After successful submission:

  • A confirmation message will be displayed.
  • The selected item information will be shown in the confirmation.
  • The order count stored in localStorage will be updated.

Accessibility Plan

The site will be designed with accessibility in mind. I will use semantic HTML elements such as header, nav, main, section, and footer. Images will include descriptive alt text. Forms will use visible labels, appropriate input types, and clear validation messages.

The color palette will be used carefully to maintain readable contrast. Important information will not be communicated by color alone. Buttons and links will have clear text, large clickable areas, and visible focus states for keyboard users. The layout will also be responsive so the site works well on both mobile and desktop screens.
